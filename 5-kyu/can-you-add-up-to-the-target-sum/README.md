<h2><a href=https://www.codewars.com/kata/64e68507cabf39dce8116872/train/typescript target="_blank">Can you add up to the target sum?</a></h2><h3>5 kyu</h3><p>Given an <code>array of numbers</code> and a <code>target number</code>, return an array of numbers that sums to the target number.</p><p>You can only pick numbers from the given array, but you can pick them as many times as you like.</p><p>To not create different solutions you should always pick as big a number as possible.</p><p>The array of numbers will be sorted in descending order (bigger numbers come first, smaller numbers later).</p><p>If it is impossible to get to the target number, return an empty array (or, depending on language, an empty value).</p><h2 id="input">Input</h2><p>As arguments you will receive:<br><code>numbers</code>: an array of numbers, positive integers from <code>1</code> to <code>1 000</code>, sorted in descending order.<br><code>target</code>: a number, a positive integer from <code>1</code> to <code>100 000</code>.<br>All inputs will be valid.</p><h2 id="output">Output</h2><p>Your function should return an array of integers from the input array that, when summed, equals the target number.<br>Return an empty array (Haskell: <code>Nothing</code>) when it is not possible.</p><h2 id="tests">Tests</h2><p>Your function should pass <code>10</code> fixed tests and <code>490</code> random tests.<br>Total number of tests is <code>500</code>.</p><h2 id="good-luck">Good luck!</h2>
