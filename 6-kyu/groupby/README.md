[ 1,9,8,4 ]</code> (yes, it's a contrived example).<br>The exception to this are methods that mutate their <code>this</code>, because in JavaScript, <code>String</code>s are immutable.</p><p>Possible syntax for this is <code>Array.prototype.groupBy.call("Mississippi")</code> ( or <code>[].groupBy.call("Mississippi")</code> or <code>[].groupBy.bind("Mississippi")()</code>, and there is an <code>apply</code> version too ), however,<br>by defining <code>String.prototype.groupBy = Array.prototype.groupBy</code> ( or <code>String.prototype.groupBy = [].groupBy</code> ) we can simply write <code>"".groupBy()</code>.<br>Because of enumerability ( which defaults to <code>true</code> for <code>=</code> but <code>false</code> for <code>Object.defineProperty</code> ), <code>Object.defineProperty(String.prototype,"groupBy",{value:[].groupBy})</code> is actually the preferred way to define the prototype method.</p><p>Any solution that works on <code>Array</code>s will probably be able to return <code>[ ["M"], ["i"], ["s","s"], ["i"], ["s","s"], ["i"], ["p","p"], ["i"] ]</code> for <code>"Mississippi".groupBy()</code>.<br>Ideally, it would return <code>[ "M", "i", "ss", "i", "ss", "i", "pp", "i" ]</code>.</p><p>Your solution will be tested for its behaviour in this case.
